Visual Studio Data Breakpoint

By 钟桓

12月 16 2015 更新日期:12月 16 2015

文章目录

有种断点称为data breakpoint， 它可用来监测某个地址的数据被写入时，也就是说该变量的值被改变，调试器会停下来。

例如，

int a = 1;
int b = 2;
b = 0;
int c = a / b;

上述代码会出现除零异常，原因就是变量b的值被写入0，为了监测b何时被写入0，可以设置一个data breakpoint，就可以对b的值进行监测。

设置的data breakpoint的方式如下：

  1. 选择“Debug—>Windows->BreakPoints”,调出“BreakPoints”窗口
  2. 在BreakPoints窗口的左上角，选择“New”，新建一个Data Breakpoint
  3. 键入监测的变量的地址，例如“&b”(监测b)，“Byte Count”32位写4，64位写8.

提示：
Data BreakPoint一般是在调试的过程中进行添加，首先在代码中添加其它断点，选在在Debug模式下运行后，才能够添加Data BreakPoint。
